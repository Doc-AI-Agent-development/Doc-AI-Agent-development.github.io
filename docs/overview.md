- 에이전트 플로우 및 활용 시나리오 구체화 (컴플라이언스, 연간계획수립, 교육자료 전처리, 교육자료 생성)

### 3. 컴플라이언스 DB 및 교육자료 저장소 스펙 정리 및 제안

실제로 동작하는 에이전트를 구현하려면 각 에이전트가 어떤 워크플로우로 동작하는지가 확정되어야 하며, 이를 위해서는 DB 및 저장소의 역할분담이 선행되어야 합니다. 이는 더미 데이터와는 별개의 문제로, 더미 데이터를 사용하더라도 에이전트가 어떤 데이터를 어디에서 어떻게 가져오는지의 컴포넌트 연결이 먼저 확정되어야 합니다.

해당 내용이 기존 공유 문서에는 정의되어 있지 않아, 법령 자문 AI 서비스의 방식을 참고하였습니다. 이를 참고하여 합리적인 컴플라이언스 DB(SK 회사의 DB를 그대로 사용하는 것으로 알고 있으나, 현재 구조를 알지 못하며 필요한 데이터가 없을 경우 테이블을 생성하면 되는 것으로 이해하였습니다)와 저장소의 설계안을 제안하고, 이를 기준으로 각 에이전트와 툴박스의 구현을 구체화하였습니다.

이미 내부에서 저장소 관련으로 합의된 내용이 있다면 공유해 주시면, 해당 내용에 맞게 구현을 변경하겠습니다.

→ 상세: [데이터 설계: 컴플라이언스 DB](./data/compliance-db.md), [교육자료 저장소](./data/content-repository.md)

# 에이전트 플로우

앞에서 정의한 [컴플라이언스 DB](../data/compliance-db.md)와 [교육자료 저장소](../data/content-repository.md)(라우터·자료 레이어)는 두 시나리오에서 다음과 같이 참조됩니다.

## 시나리오 A. 연간계획 수립

컴플라이언스 DB만 쓰고 교육자료 저장소는 건드리지 않습니다. 무슨 교육을 언제 할지를 정하는 일이지 내용을 만드는 것이 아니기 때문입니다.

1. **실시 의무 조회**: 컴플라이언스 에이전트가 담당자에게 할당된 팀이 실시해야 할 교육 목록과 각각의 주기·법정시간·근거를 가져옵니다.
2. **계획 구성**: 연간교육 계획 에이전트가 그 목록을 일정표로 배치합니다. 어떤 교육을 언제 몇 시간 실시할지가 계획의 뼈대가 됩니다.
3. **검토·확정**: 담당자가 초안을 보고 운영 항목(강사·방법 등)을 채워 확정합니다. 결재는 기존 USHE 절차를 따릅니다.

## 시나리오 B. 교육자료 생성

세 계층이 모두 쓰이는 흐름입니다. (0번) 담당자가 특정 교육자료 제작을 요청하며 시작합니다.

1. **기준 로드**: 컴플라이언스 에이전트가 그 교육의 필수항목(체크리스트)·주기·법정시간·시험유형·구분(법정/절차/법정외)을 가져옵니다.
2. **아젠다 골격**: 필수항목을 순서대로 펼쳐 목차의 뼈대로 삼습니다. 자료 구조가 법정 기준에서 출발합니다.
3. **문서 범위 확보**: 라우터 레이어에서 그 교육에 필요한 문서 목록을 가져옵니다. 이것이 자료 수집의 범위가 됩니다.
4. **항목별 내용 채우기**: 교육자료 전처리 에이전트가 자료 레이어에서 항목별 내용을 가져옵니다. 문서가 작고 단일 주제이거나 골든 문서로 지정돼 있으면 원문을 그대로 읽고, 크거나 여러 주제가 섞였으면 범위 안에서 의미검색으로 관련 부분만 꺼냅니다. 매핑 정보가 부족하다고 판단되면 추가 벡터 검색을 진행할 수 있습니다.
5. **중간 점검**: 요건 검사 에이전트가 모은 내용이 필수항목을 다 덮었는지 검사합니다. 빈 항목이 있으면 4단계로 돌아가 더 넓게 검색하거나, 내부 자료를 요청하거나, 구멍으로 표시합니다.
